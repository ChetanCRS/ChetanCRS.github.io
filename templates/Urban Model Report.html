Project Report: Compact Urban Model

A Python program to model simplified city grids and demonstrate how layout changes affect neighborhood accessibility.

Project Goal

This project was built to explore a core question in urban planning: how can we measure the quality of a city's layout? My goal was to create a simple yet powerful tool to quantify how planning decisions impact the daily lives of residents.

Using a simplified grid-based representation of a city, the model calculates an "Accessibility Score" to show how walkable and convenient a neighborhood is. It's designed to be a clear and direct way to test planning ideas.

My Approach

The model's logic is straightforward and built on a few key ideas.

1. The Grid System

The city is modeled as a 2D grid. Each cell is a parcel of land that can be assigned a specific type, like a building, a park, or a road. This makes it easy to visualize the layout and for the program to process the spatial relationships.

2. The Proximity Algorithm

The main function of the program is to calculate accessibility. It checks each residential building to see if it is within a reasonable walking distance of essential amenities. The final score is the percentage of homes that are well-served, giving a clear metric for how good the layout is.

Case Study: Centralization vs. Dispersion

To show the model in action, I ran a simple comparison. I created two city layouts with the exact same number of buildings. The only variable was the placement of the amenities and commercial zones.
